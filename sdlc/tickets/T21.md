## T21 — Configure Python logging so output appears in `docker compose logs`
git branch: `ai-agent/t21_configure_logging`

PRD reference: §7.5 Application Logging, §9 Configuration (`LOG_LEVEL`), SC9.

**Root cause:** Python's `logging` subsystem discards all log records by default when no handler is attached to the root logger. The application uses `logging.getLogger(__name__)` throughout (per T1 acceptance criterion 1.C), but never calls `logging.basicConfig()` or otherwise attaches a `StreamHandler`, so every `logger.info()` / `logger.warning()` / `logger.error()` call is silently dropped. Running `docker compose logs -f` therefore shows no application output.

### Reference implementation

Add the following block to `app/main.py` immediately after the import statements:

```python
import logging
import sys
from app.config import LOG_LEVEL

logging.basicConfig(
    level=logging.getLevelName(LOG_LEVEL),
    format="%(asctime)s %(levelname)-8s %(name)s — %(message)s",
    stream=sys.stderr,
)
```

And in `app/config.py`, add alongside the existing variables:

```python
LOG_LEVEL: str = os.environ.get("LOG_LEVEL", "INFO").upper()
```
