## T23 — Optimize Stage 3 training sample generation to prevent page cache flooding
git branch: `ai-agent/t23_optimize_training_sampling_stage3`

PRD reference: §7.4, §12 (Memory pressure during index build)

**Problem:** 
In `app/build_index.py::build_faiss_index()`, the codebase currently does the following:
1. When loading a numpy file matching `npy_magic`, it runs `np.load(str(embeddings_path)).astype("float32")`, which loads the entire 15+ GB embeddings array into RAM.
2. When training the IVF index, it randomly samples vector indices and fetches them from the memory-mapped array:
   ```python
   sample_indices = rng.sample(range(n_titles), min(n_sample, n_titles))
   train_vectors = all_vectors[sample_indices].astype("float32")
   ```
   `all_vectors[sample_indices]` triggers advanced indexing in NumPy, creating an in-memory copy of the sampled vectors (1.5 GB). 
   More importantly, because the indices are random and span the entire 15.36 GB file, the OS is forced to random-access almost every page of the memmap file from disk. This floods the page cache, bloating RSS to the size of the entire file and causing severe disk thrashing on low-memory servers.

**Solution:**
1. If loading a `.npy` file, memory-map it via `np.load(..., mmap_mode="r")`.
2. Avoid redundant `.astype("float32")` calls on arrays that are already of type `float32`.
3. Gather training vectors using sequential/strided slicing (basic indexing) which returns a view and reads pages sequentially:
   `stride = max(1, n_titles // n_sample)`
   `train_vectors = all_vectors[::stride][:n_sample].copy()`
   This sequential reading maximizes disk read throughput and allows the OS page cache to evict read pages efficiently, preventing RSS ballooning and disk thrashing.

### Reference implementation

In `build_faiss_index()`:
```python
    if header_magic == npy_magic:
        # File written via np.save() — preserves shape in header.
        all_vectors = np.load(str(embeddings_path), mmap_mode="r")
        if all_vectors.dtype != np.float32:
            all_vectors = all_vectors.astype("float32")
```

And replacing the sample generator section:
```python
    # ── train IVF quantizer on a sequential stride sample ───────────────── #
    n_sample = int(n_titles * sample_frac)
    n_sample = min(n_sample, n_titles)
    
    if n_sample > 0:
        stride = max(1, n_titles // n_sample)
        # Slicing with stride returns a view; copying it creates a contiguous C-array
        # by reading the file sequentially rather than performing random access.
        train_vectors = all_vectors[::stride][:n_sample].copy()
    else:
        train_vectors = np.empty((0, embed_dim), dtype="float32")
```
