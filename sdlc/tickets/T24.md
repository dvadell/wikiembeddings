## T24 — Support Scalar Quantization (SQ8) to reduce index RAM footprint by 4x
git branch: `ai-agent/t24_support_scalar_quantization`

PRD reference: §7.1, §9 (FAISS_INDEX_TYPE), §12 (Memory pressure during index build)

**Problem:** 
The FAISS index type currently built is `IVF4096,Flat` (`faiss.IndexIVFFlat`). This index stores all vectors in their raw 32-bit float representation. For 10 million Wikipedia titles embedded with 384-dimension vectors, the index alone consumes ~15.36 GB of RAM. This is the single largest consumer of RAM at the end of the build phase and during serving.

**Solution:**
Add a new configuration setting `FAISS_INDEX_TYPE` (default: `"IVFFlat"`). Support `"IVFSQ8"` (8-bit Scalar Quantization).
When `FAISS_INDEX_TYPE` is set to `"IVFSQ8"`, build the index using `faiss.IndexIVFScalarQuantizer` with `faiss.ScalarQuantizer.QT_8bit` instead of `faiss.IndexIVFFlat`. 
An 8-bit Scalar Quantizer stores each vector element as a 1-byte integer instead of a 4-byte float. This reduces the index memory footprint from 15.36 GB to 3.84 GB, saving ~11.5 GB of RAM during build and serving.

At query/load time, `faiss.read_index()` automatically detects the index format, so the search endpoint inside `app/main.py` requires no changes.

### Reference implementation

In `app/config.py`:
```python
# FAISS index type: "IVFFlat" (exact vectors, more RAM) or "IVFSQ8" (8-bit scalar quantization, 4x less RAM)
FAISS_INDEX_TYPE: str = os.environ.get("FAISS_INDEX_TYPE", "IVFFlat").upper()
```

In `app/build_index.py` (`build_faiss_index`):
```python
    # ── resolve index type ──────────────────────────────────────────── #
    index_type = index_type.upper() if index_type else "IVFFLAT"

    quantizer = faiss.IndexFlatL2(embed_dim)
    if index_type == "IVFSQ8":
        index = faiss.IndexIVFScalarQuantizer(
            quantizer, embed_dim, int(nlist), faiss.ScalarQuantizer.QT_8bit, faiss.METRIC_INNER_PRODUCT
        )
    else:
        index = faiss.IndexIVFFlat(
            quantizer, embed_dim, int(nlist), faiss.METRIC_INNER_PRODUCT
        )
```

In `app/build_index.py` (`start_pipeline`):
```python
        build_faiss_index(
            str(embeddings_path),
            str(_cfg.TITLES_FILE),
            Path(str(_cfg.FAISS_INDEX)),
            str(_cfg.BUILD_MANIFEST),
            int(_cfg.BUILD_NLIST),
            float(_cfg.BUILD_SAMPLE_FRAC),
            bool(_cfg.BUILD_RESUME),
            lambda v: mprogress("faiss", v),
            index_type=str(_cfg.FAISS_INDEX_TYPE),  # Pass config index type
        )
```
